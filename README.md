# Social Games — Host Handbook

> WhatsApp-based social games for groups of 5-8 players.
> Fully remote. Fully async. No apps, no downloads, no real-time coordination needed.

---

## What Is This?

A collection of ready-to-host social games that run entirely inside WhatsApp. Each game lasts 5-7 days and is designed for people who can't be online at the same time — every action has a multi-hour window.

Anyone with this handbook can host a complete session of any game.

## How Async Works

These games are built for people who take time to reply. No one needs to be glued to their phone.

**Core principles:**

1. Every action has a **multi-hour window** (minimum 6 hours, usually 8-12)
2. Group chat is for discussion — no one is penalized for chatting less
3. All structured actions (votes, submissions) go via **private DM** to the host
4. A daily summary keeps slow responders caught up without reading every message
5. Missing one window is a warning, not a death sentence

**Timing defaults:**

| Phase Type | Minimum Window | Recommended |
|------------|---------------|-------------|
| Submissions (votes, actions) | 6 hours | 8-12 hours |
| Discussion | 12 hours | 18-24 hours |
| Challenge completion | 4 hours | 6-8 hours |
| Late game (final 2 days) | 4 hours | 6 hours |

---

## Handling Inactivity

Same rules across all games:

| Missed Windows | Action |
|----------------|--------|
| 1 | Host DMs a friendly reminder. No penalty. |
| 2 consecutive | Their action defaults (abstain/skip). Group told "[Player] was absent." |
| 3 consecutive | Player is removed. Role neutralized. Game continues. |

## Hosting Checklist (Before Any Game)

- [ ] Recruit 5-8 players
- [ ] Create WhatsApp group
- [ ] Decide duration (5 or 7 days) and set start date
- [ ] Read the full game file and prepare materials (challenges, clues, missions)
- [ ] Assign roles via private DM
- [ ] Post the welcome message
- [ ] Set daily alarms for your host duties

## Daily Host Routine (All Games)

| Time | Action |
|------|--------|
| Morning (09:00) | Post daily challenge/prompt/phase announcement |
| Midday (13:00) | Check for inactive players. Send reminder DM if needed. |
| Evening (20:00-22:00) | Collect submissions. Process votes/actions. Resolve outcomes. |
| Night (22:00-23:00) | Post results. DM next-day info to special roles. |

## Handling Common Problems

**Player goes silent for 24+ hours:**
> DM them: "Hey [name], checking in — you have [X hours] left to submit your [thing] or it defaults to skip. Let me know if you need more time!"

**Player wants to quit mid-game:**
> Remove gracefully. Don't reveal their role. Say "[Player] had to leave for personal reasons." Reassign key roles secretly if needed, or shorten the game by 1 day.

**Two players have real conflict:**
> DM both privately. Never address in group. Offer to pause or adjust. These are games — keep them fun.

**Game feels slow/dead:**
> Post a provocative prompt. Or compress: "Slow day — tonight's window extended 4 hours. Tomorrow runs on accelerated schedule."

## Design Philosophy

- **No real-time sync required** — every window is hours, not minutes
- **No app downloads** — WhatsApp only
- **No complex rules to memorize** — each player gets only what they need via DM
- **Drama from social dynamics** — the rules create situations; the players create stories
- **Everyone stays engaged** — most games have no elimination, and those that do give eliminated players a role (jury, spectator with influence)
